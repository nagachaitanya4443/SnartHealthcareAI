# 🏥 Smart Healthcare Analytics

An AI-powered healthcare analytics dashboard built with **Streamlit**. It predicts
diabetes, heart disease, and chronic kidney disease risk from patient data,
tracks patient records, visualizes population analytics, and generates PDF
health reports — with a built-in AI health assistant chatbot.

## ✨ Features

- **Login / Sign-up** — simple username/password auth (SQLite-backed)
- **Patient Management** — add, search, view, and delete patient records
- **Disease Prediction** — RandomForest models for diabetes, heart disease, and CKD
- **Risk Analysis** — composite multi-disease risk score with a gauge chart and recommendations
- **Analytics** — prediction-history trends and dataset exploration (correlation heatmaps, distributions)
- **AI Assistant** — offline rule-based Q&A, or live Claude-powered answers if you add your own API key
- **Report Generator** — downloadable PDF patient reports (via ReportLab)

## 🚀 Getting Started

1. **Install dependencies**
   ```bash
   pip install -r requirements.txt
   ```

2. **(Already done for you)** Generate datasets & train models — only needed again if you replace the CSVs:
   ```bash
   python train_models.py
   ```

3. **Run the app**
   ```bash
   streamlit run app.py
   ```

4. **Log in** with the pre-seeded demo account:
   - Username: `admin`
   - Password: `admin123`

   Or create your own account from the Sign Up tab.

## 🧠 About the Data & Models

The bundled `datasets/*.csv` are **synthetically generated** (see `train_models.py`)
so the whole app works immediately without needing external data. The synthetic
data mimics the structure of well-known public datasets:

- `diabetes.csv` → Pima Indians Diabetes-style features
- `heart.csv` → UCI Heart Disease-style features
- `kidney.csv` → UCI Chronic Kidney Disease-style features (numeric subset)

**For real-world / production use**, replace these CSVs with real, properly
licensed clinical datasets, then re-run `python train_models.py` to retrain
the `.pkl` models on real data. Do not use the synthetic models for actual
clinical decision-making.

## 🤖 AI Assistant

By default, the AI Assistant page uses a small offline keyword-based
knowledge base — no API key required. To get richer, general-purpose
answers, paste your own Anthropic API key into the sidebar field on that
page. The key is only kept in your local Streamlit session and is never
persisted to disk.
